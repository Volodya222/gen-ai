# Ультиматум: homo silicus — воспроизводится ли человеческое отвержение у LLM?

Финальный проект, **трек A (исследовательский), сценарий 3 «Агенты в рынке» (по Хортону).**

Симулируем классическую игру **«Ультиматум»**: Предлагающий-агент делит 100 ₽,
синтетическая персона-Отвечающий принимает или отвергает. Проверяем гипотезу:
**воспроизводит ли LLM ключевой стилизованный факт экспериментальной экономики —
систематическое отвержение низких предложений** (нарушение теоретико-игрового
равновесия).

## Одна команда запуска

```bash
pip install -r requirements.txt && cp .env.example .env   # впиши свой DeepSeek-токен в .env
python make_input.py && python eval.py
```

`eval.py` прогонит 15 тестовых входов через пайплайн и положит артефакты в `output/`.

### Самопроверка кода без токена (офлайн-симуляция)

```bash
python make_input.py && python eval.py --offline && python test_detector.py
```

## Что где лежит

| Файл | Назначение |
|------|-----------|
| `llm_client.py` | OpenAI-совместимый клиент из курса (`make_client`, `response_model`). Добавлен учёт `usage`. |
| `schema.py` | Pydantic-схемы + `field_validator` (доля 0..100, оценка судьи 1..5). |
| `personas.py` | 5 синтетических персон Отвечающего с демографией и ground-truth порогом. |
| `tools.py` | Инструменты агента: калькулятор выплат `expected_payoffs`, `settle_round`, `acceptance_stats`. |
| `prompts.py` | Системные промпты Предлагающего, Отвечающего, судьи. |
| `pipeline.py` | Ядро: мультиагент + персоны + tools + LLM-as-judge + детектор галлюцинаций. |
| `make_input.py` | Генерация `input/test_cases.json` (15 входов) и `input/human_benchmark.json`. |
| `eval.py` | Eval: правильность (exact-match, judge) + путь (шаги, инструменты, токены, стоимость) + проверка гипотезы. |
| `simulate.py` | Свободная мультиагентная симуляция Предлагающий↔персона, транскрипт. |
| `test_detector.py` | Юнит-тест детектора галлюцинаций. |
| `input/` | Тест-кейсы и человеческий эталон. |
| `output/` | Артефакты прогона: `eval_results.json`, `eval_table.md`, `failures.json`, транскрипты. |
| `отчёт.md` | Отчёт по проекту. |

## Техники курса в проекте

1. **Синтетические персоны** — Отвечающий обусловлен демографическим профилем (`personas.py`).
2. **Агент с инструментами** — Предлагающий зовёт калькулятор выплат перед ходом (`tools.py`).
3. **Мультиагент** — Предлагающий и Отвечающий взаимодействуют в цикле (`pipeline.py`, `simulate.py`).
4. **LLM-as-judge** — отдельная модель оценивает согласованность хода (`judge_move`).
5. **Структурированный вывод** — все ходы через `response_model` + `max_retries` + `field_validator`.
6. **Проверка галлюцинаций** — детектор ghost-раундов и выдуманных чисел (`detect_hallucinations`).
